# SMSForwarder Client

基于 [JSBox](https://xteko.com) 的 **SmsForwarder 远程控制客户端**。通过 Wi‑Fi 或内网穿透，在 iPhone / iPad 上远程操作已开启「主动控制·服务端」的 Android 设备。

> 服务端需安装 [SmsForwarder](https://github.com/pppscn/SmsForwarder) **v3.0.0** 及以上，并开启对应远程 API。  
> 接口说明见官方文档：[附录2：主动请求（远程控制）](https://github.com/pppscn/SmsForwarder/wiki/%E9%99%84%E5%BD%952%EF%BC%9A%E4%B8%BB%E5%8A%A8%E8%AF%B7%E6%B1%82(%E8%BF%9C%E7%A8%8B%E6%8E%A7%E5%88%B6))

---

## 功能

首页顶部信息卡片会显示服务端设备备注、SIM 卡备注及已开启的远程能力；下方为功能入口：

| 功能 | 说明 |
|------|------|
| 发短信 | 指定 SIM 卡槽向一个或多个号码发送短信 |
| 查短信 | 按类型（接收/发送）、关键字分页查询 |
| 查通话 | 按类型、号码分页查询通话记录 |
| 查话簿 | 按姓名或号码模糊查询联系人 |
| 加话簿 | 向 Android 通讯录添加联系人 |
| 查电量 | 查看电量、充电状态、健康度等 |
| 远程 WOL | 通过 MAC 地址唤醒局域网设备 |
| 查定位 | 获取设备最近一次 GPS 定位 |

## 使用前准备（Android 端）

1. 在备用机上安装并打开 **SmsForwarder**，进入 **主动控制 · 服务端**。
2. 记下服务端地址，例如：
   - 局域网：`http://192.168.1.100:5000`
   - 内网穿透：`http://your-domain.example.com`
3. 在服务端设置中 **开启** 你需要使用的远程 API（发短信、查短信等）。
4. 若启用了 **签名校验**，记下签名密钥（Secret）。
5. 确保 JSBox 所在设备能访问上述地址（同一 Wi‑Fi，或穿透可达）。

---

## 首次设置（JSBox 端）

1. 在 JSBox 中运行本脚本。
2. 点击右上角 **设置**（齿轮）。
3. 填写 **服务端地址**（完整 URL，无需末尾斜杠），例如 `http://192.168.1.100:5000`。
4. 若服务端开启了签名校验：
   - 打开 **校验签名**；
   - 填写 **签名密钥**（与服务端一致）。
5. 点击 **测试连接**，确认提示连接成功。
6. 返回首页，顶部信息卡片应显示设备备注与已开启功能；点击卡片可刷新配置。

未配置服务端地址时，首页会显示占位提示。

---

## 使用说明

- **首页**：查看服务端状态；点击功能方块进入对应操作页。
- **功能页**：填写参数后点击查询/发送，结果展示在页面底部。
- **设置 · 接口文档**：跳转 SmsForwarder 官方 API 说明。
- 请求失败时，请检查：地址是否正确、网络是否互通、对应 API 是否在服务端已开启、签名配置是否一致。

---

## 环境要求

- JSBox SDK ≥ 2.19.0
- iOS / iPadOS ≥ 14.0
- SmsForwarder 服务端 ≥ 3.0.0
